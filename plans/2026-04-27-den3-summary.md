## 1. Структурное саммари по блокам

### 🔹 GitHub и откаты

**GitHub** — облако для кода (как Яндекс.Диск, но умнее). Хранит историю всех изменений.

**Ключевые понятия:**
- **Репозиторий** — папка проекта в облаке (публичный или приватный)
- **Commit** — сохранение изменений с описанием («сохранили фичу X»)
- **Push** — отправка коммита с локального компьютера в GitHub
- **Deploy** — отправка кода «в жизнь» (на хостинг, чтобы был доступен извне)

**Зачем нужен:**
- Код в облаке — компьютер сломался, код цел
- Откат — Cline что-то сломал → смотришь последний коммит → откатываешь
- Безопасность — история всех изменений

**Как работать:** говоришь Cline «создай приватный репозиторий на GitHub, назови X, сделай первый комит и запуш» — всё остальное он делает сам (git init, add, commit, push).

**Откат:** «Удали главную страницу» → он удалил → «Откати последние изменения» → вернулась.

### 🔹 Деплой: Vercel / Railway / Amvera

**Деплой** — отправка проекта в жизнь, чтобы он был доступен через ссылку извне. Без деплоя бот/сайт работает только пока включён ваш компьютер.

**Международный путь — Railway (railway.com):**
- Всё в одном месте: сайт, приложение, PostgreSQL, Redis, MongoDB
- Подключил GitHub → автодеплой
- Можно дать Cline ключи удалённого доступа → он сам всем управляет
- Cline: «Помоги мне полностью пройти путь, чтобы сайт оказался доступен через Railway»

**Бесплатный — Vercel:**
- Бесплатный хостинг для статики и простых приложений
- Cline может сам отправить проект и дать ссылку
- Для тестов и MVP — идеально

**Российский путь — Amvera Cloud:**
- Серверы в России (ФЗ-152, персональные данные)
- Есть PostgreSQL, Redis
- Проксирование AI-запросов (Claude/OpenAI не отбивает с российских серверов)
- Поддержка OpenRouter
- Бесплатный тест: 111 руб. на баланс
- Аналог Railway, но для РФ

**Важно:** если проект для российских пользователей и обрабатывает персональные данные → серверы в России + уведомление в Роскомнадзор.

### 🔹 Chrome Claude и связка с Cline

**Chrome Claude** — расширение Claude в Google Chrome. Магический инструмент для автоматизации рутинных задач.

**Как работает связка VS Code (Cline) + Chrome Claude:**
1. Cline пишет код и говорит: «Тебе нужно зайти сюда, нажать сюда, получить API-ключ»
2. Ты отвечаешь: «Напиши промт для Chrome Claude, который всё это сделает за меня»
3. Cline пишет промт → копируешь → вставляешь в Chrome Claude
4. Chrome Claude сам регистрируется, настраивает, получает ключи
5. Отдаёт тебе ключи → скармливаешь обратно Cline

**Что можно автоматизировать через Chrome Claude:**
- Регистрацию на сервисах (Railway, Amvera, платёжках)
- Настройку Яндекс.Метрики, Вебмастера
- Получение API-ключей
- Любую рутину в браузере

**Принцип:** ничего не делать руками. Cline пишет промт для Chrome Claude, Chrome Claude выполняет.

### 🔹 База данных

**PostgreSQL** — «Excel-таблица в облаке». Хранит данные: пользователи, имейлы, заказы, оплаты, результаты проверок.

**Redis** — «оперативная память для сайта». Ускоряет, кэширует, выстраивает очередь.

| | PostgreSQL | Redis |
|---|---|---|
| Аналог | Морозильник — хранит надёжно и надолго | Стол — мгновенно то, что нужно сейчас |
| Для чего | Постоянное хранение данных | Кэширование, очередь задач |
| Когда нужна | Регистрация, личный кабинет, оплата | Высокие нагрузки, рассылки |

**Когда БД нужна:** регистрация, сохранение данных, оплата, личный кабинет
**Когда НЕ нужна:** простой лендинг без форм, прямая отправка лидов в Telegram-бот

**Как просить:** «Подбери стек» → Cline сам выберет → «Давай разработаем архитектуру БД» → он нарисует таблицы, связи, типы данных.

**Безопасность:** защита по IP (1 IP = макс 10 проверок в час), Security Review после каждой фичи.

### 🔹 Дизайн: 4 подхода

**Подход 1 — Скриншот (кража дизайна):**
- Заходишь на сайт, который нравится
- Скриншот → скидываешь Cline → «Укради этот дизайн»
- Pinterest: «дизайн CRM системы» → скрин → Cline

**Подход 2 — Библиотеки компонентов:**
- **Aceternity UI** — 200+ анимированных компонентов, open source (3D-карточки, параллаксы, свечения)
- **Magic UI** — аналогичный сервис для карточек
- **Framer Motion** — для анимации
- **Animate.css** — простые CSS-анимации
- Просто сказать: «Установи Aceternity UI и Magic UI» → он скачает и интегрирует

**Подход 3 — CSS-кража (глубокая):**
- F12 → Elements → скопировать стили
- Cline поднимает цвета, архитектуру, всё понимает
- Лучше всего перетаскивает дизайн, сделанный кодом (CSS), а не картинками

**Подход 4 — Промт-дизайнер:**
- «Установи Aceternity UI и Magic UI. Полностью перерабатывай дизайн: hero-секции с эффектами, карточки с анимацией, кнопка оплаты со свечением, анимация загрузки»
- Лучший промт: «Что-то некрасиво. Сделай красиво» → итерации

**Ключевая мысль:** дизайн не для красоты, а для конверсии. «Симпатично выглядит, но работает для нас». Не «Cline, сделай красиво», а осознанный выбор библиотек + референсы.

### 🔹 Оплаты: 2 ветки (5 шагов)

**Единая методология для ЛЮБОЙ платёжной системы — 5 шагов:**

1. **Найди SDK** — «Найди официальный NPM-пакет и документацию API у [платёжка]»
2. **Изучай** — «Изучай документацию, разберись в интеграции»
3. **Реализуй** — «Реализуй полностью, чтобы оплата [сумма] работала»
4. **Ключи через Chrome Claude** — «Напиши промт для Chrome Claude, который зарегистрируется, настроит и получит ключи за меня»
5. **Безопасность** — все ключи ТОЛЬКО в `.env` / Environment Variables, никогда в коде

**Платёжки для России:**
- **Cloud Payments** (использует спикер) — САС, подписки
- **Робокасса** — универсальная
- **Тинькофф (Tinkoff)** — эквайринг

**Международные:**
- **Stripe** — международные платежи

**Для подключения платёжки нужно:** ИП или самозанятость. Нельзя быть «никем».

### 🔹 Методология проектов

**Папка `.business/`** = второй мозг. Контекст, который Cline читает перед каждой задачей. Один проект = одна папка `.business/`.

**Архитектура проекта:**
1. Определил идею → «Подбери стек» → Cline выбрал технологии
2. «Сделай план» → план в `plans/` с фазами
3. Реализация по фазам → каждая фича = commit
4. После каждой фичи: **Security Review**
5. Деплой → проект в жизни

**Принцип:** каждый прожитый день должен работать на тебя завтра. Одна функция = один день работы = работает каждый день.

**Security Review** — после КАЖДОЙ фичи говоришь: «Сделай Security Review». Cline сам находит уязвимости и рекомендует исправления.

**Автопилот (шаблон старта):** будет размещён на GitHub как open source — скачиваешь, даёшь ссылку Cline, он запускает автопилот, который сам делает всю домашку 3 дней (создаёт папку `.business/`, заполняет `cline.md`, настраивает `settings.json`).
