# plans/

Папка для технических планов реализации функций проекта.

## Правила

1. **Один план = одна функция.** Каждый файл описывает ровно одну фичу.
2. **Формат имени файла:** `YYYY-MM-DD-название-функции.md`
   - Пример: `2026-04-26-user-auth.md`
3. **Структура плана — фазы.** Каждая фаза имеет статус:
   - `[ ]` — не начата
   - `[x]` — выполнена
4. **Итоговый блок** в конце плана:
   - Реализован ли план целиком?
   - Что осталось невыполненным и почему?
5. **Любой агент**, работающий с планом, обязан его актуализировать:
   - отмечать выполненные фазы,
   - дописывать, что было сделано,
   - обновлять итоговый блок.

## Пример структуры плана

```markdown
# Название функции

## Цель
Краткое описание, что нужно реализовать.

## Фазы
- [ ] Фаза 1: Описание
- [ ] Фаза 2: Описание
- [ ] Фаза 3: Описание

## Итог
- Статус: не начат / в процессе / завершён
- Выполнено: —
- Осталось: —
